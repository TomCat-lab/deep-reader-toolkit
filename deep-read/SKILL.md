# Deep Read - 深度阅读

用《学会提问》的批判性思维框架拆解任何书籍或资料，核心流程：**沉淀作者经验 → 批判性思维提炼 → 精华内化**。

## 输入

用户提供文件路径，支持 PDF、EPUB、TXT、Markdown 格式。

## 文件预处理

如果文件不是 .md 格式，必须先转换为 Markdown 再阅读。

### 方式一：Bash 可用时，自动调用 CLI

```bash
cd <skill目录>/deep-read && uv run python cli.py "<文件路径>" "<输出目录>/<书名>.md"
```

### 方式二：Bash 不可用时，提示用户手动执行

请用户在终端运行：

```powershell
cd <skill目录>/deep-read; uv run python cli.py "<文件路径>" "<输出目录>\<书名>.md"
```

> 依赖安装：`pip install -r requirements.txt` 或 `uv pip install -r requirements.txt`

转换完成后，用 Read 工具读取生成的 .md 文件。

## 流程

### 阶段 1 — 快速扫描

1. 读取 Markdown 文件，提取目录结构和核心内容
2. 识别材料类型（论说文、学术论文、商业书籍、科普、教材等）
3. 输出简要扫描报告，包含：
   - 书名、作者（含职业身份和角色分工）、材料类型、核心工作成果
   - 核心论题和结论的初步判断
   - 章节概览（含结构图和详情列表）
4. 询问用户是否需要深度分析，以及想深挖哪些章节或步骤

#### 章节概览格式

**上方：结构图**——展示章节间的分组和逻辑关系。根据材料内容识别分组（如认知偏误类、信息处理类、社会互动类等），用分组+箭头展示递进关系。

**下方：详情列表**——每章包含4个字段：
- 章节名 + 一句话摘要
- 最相关步骤（1-2个批判性思维步骤）
- 核心证据/案例（该章最关键的论据）

### 阶段 2 — 作者论证链

按作者自己的论证逻辑串联关键信息，帮助读者完整理解作者的思想体系。这是"沉淀作者经验"的核心环节。

输出一个可交互的论证链，包含7个节点：

1. **观察**：作者观察到了什么现象？原始事实是什么？（先不判断错误的性质）
2. **分类**：这个现象属于什么类型？关键判断：错误是随机的还是系统性的？只有确认了错误类型，才能追问"为什么"。**必须展示分类的证据（如对照实验、基线比较）。必须标注：样本覆盖了哪些人群/国家，遗漏了哪些。如果证据的覆盖范围与作者的归因范围不匹配（如只测了发达国家却归因于"人类"），必须明确指出。**
3. **求证**：作者如何从分类推导到原因？**必须展示假说→证伪→修正的迭代过程。** 作者通常不会一步到位找到原因，而是先尝试更简单的假说（如"知识过时"），被证伪后才修正为更深层的解释（如"本能"）。这个迭代过程是论证的关键环节，不能跳过。
4. **原因**：为什么会发生这种类型的错误？归因于什么？因果链是什么？**每个原因必须展示运作机制：人们依据这个原因怎么认识现实世界→这个认识方式在什么条件下会导致系统性错误→具体的错误路径。不能只给定义，必须展示"从本能到误判"的因果链。**
5. **现实展开**：第4步列出了原因，这一步展示每个原因在现实中的具体表现——人们根据这个原因具体做了什么、说了什么、想了什么，这些现象又对现实造成了什么影响。全部从文本提取，不脑补。详见下方"现实展开操作规程"。
6. **思维工具**：作者提出什么实践方法/思维工具？**必须展开：提取每种工具的核心思想和核心动作（具体怎么做），从原文提取，不脑补。不能只写一句话概括。**
7. **结论**：直接呈现作者的原始结论，不重构。**必须检查：结论的归因范围是否与证据的覆盖范围匹配。**

每个节点可点击展开详情，包含原文引用和关键数据。

#### 现实展开操作规程

现实展开的核心任务：**对第4步的每个原因，从文本中提取它在现实中产生的所有具体表现（现象），以及这些现象对现实造成的影响。**

**职责**：①对每个原因提取现象（人们根据这个原因具体做了什么、说了什么、想了什么——不是对原因的概括复述）；②对每个原因提取影响（这个现象导致了什么后果/危害）；③全部从文本提取，不脑补。

**关键区分**：现象≠原因的定义复述。例如"一分为二"的现象不是"人们把世界分为两类"（这是定义），而是"记者只报道对立和极端""学生说他们永远不可能像我们这样生活""世界银行花了17年才改变分类定义"等具体表现。

##### 第一步：逐原因提取

对第4步列出的每个原因，从文本中提取两层信息：

1. **现象**：人们根据这个原因具体做了什么、说了什么、想了什么？书中描述的所有具体表现都要列出，不能只写一条概括。现象≠原因的定义复述。
2. **影响**：这个现象对现实造成了什么后果？改变了什么行为、决策或结果？

- 必须引用书中原文作为依据
- 只提取作者说出的现象和影响，不脑补
- 如果作者只描述了现象未说明影响，标注"作者未说明影响"

##### 第二步：输出现实展开表

| 原因 | 现象（原文提取） | 影响（原文提取） |
|------|-----------------|-----------------|
| 原因A | 现象描述+原文引用 | 影响描述+原文引用 |
| 原因B | ... | ... |

##### 现实展开完整输出

1. 每个原因的现象（含原文引用）
2. 每个原因的影响（含原文引用）
3. 现实展开表

**注意**：此阶段严格按作者的逻辑组织，不引入批判性思维框架的视角。目的是让读者先完整理解作者说了什么，再进入批判性分析。

### 阶段 3 — 分析路线图

将"适用范围评估"、"依赖链"和"13步深度分析"合并为一个导航+内容双层的交互式板块。

#### 路线图导航层

展示13步框架的依赖链和步骤列表，每个步骤可点击展开分析内容：

```
核心链（严格有序，后步依赖前步）：
2 论题和结论 → 3 理由 → 5 假设 → 6 谬误 → 7/8 证据效力 → 9 替代原因 → 10 数据欺骗性 → 11 省略信息 → 12 合理结论

独立步骤（可随时插入）：
1 提问方式、4 歧义词、13 认知障碍
```

**回溯规则**：所有判断都是暂时的。当后续步骤的发现要求修正前步判断时，必须回溯并修正，在报告中记录修正内容。

#### 内容层

点击路线图中的步骤节点，展开该步骤的深度分析。每步包含：
- 核心问题
- 操作方法
- 分析内容（含原文引用）
- 修正记录（如有回溯修正）

#### 批判性思维13步框架

##### 步骤 1：提问方式

**核心问题**：作者的提问方式是什么？属于海绵式还是淘金式思维？

**操作方法**：
1. 检查作者是否只呈现信息（海绵式），还是主动质疑并引导读者互动（淘金式）
2. 判断作者是否使用强势批判性思维（评价所有信念包括自己的）还是弱势批判性思维（只捍卫已有立场）
3. 检查作者是否在论证过程中修正过自己的结论——这是强势批判性思维的标志

**适用材料类型**：全部

---

##### 步骤 2：论题和结论

**核心问题**：作者的核心论题和结论分别是什么？是描述性还是规定性？

**操作方法**：
1. 先找论题：检查标题、开头段落、作者背景
2. 判断论题类型：描述性（是什么）vs 规定性（该怎样）
3. **防混淆规则：论题类型≠结论类型。论题类型只取决于它提出的问题类型，不取决于回答的类型。** 例如："为什么人们会犯错？"是描述性论题，即使结论包含"应当如何改正"这样的规定性内容，论题仍然是描述性的。论题和结论必须分别独立判断类型。
4. 找结论的5条线索：
   - 线索一：问"论题是什么"，结论是对论题的回应
   - 线索二：找指示词（因此、由此可知、表明、证明、事情的真相是）
   - 线索三：在可能的位置查看（开头、结尾）
   - 线索四：排除不可能作为结论的内容（例证、数据、定义、背景材料、证据）
   - 线索五：检查交流语境和作者背景
4. 如果论题未明说，先找结论再反推论题

**适用材料类型**：论说类、学术类、科普类、商业类

---

##### 步骤 3：理由

**核心问题**：支撑结论的理由和证据有哪些？

**操作方法**：
1. 先问"为什么作者相信这个结论"
2. 通过提示词寻找理由（因为、由于、鉴于、理由是、证据是、首先…其次）
3. 区分理由和证据：理由是原理/逻辑，证据是事实/数据
4. 画出论证结构图：结论←理由1←证据A，理由2←证据B…
5. 检查理由是否完整——是否遗漏了关键支撑

**适用材料类型**：论说类、学术类、科普类、商业类

---

##### 步骤 4：歧义词

**核心问题**：关键术语是否有歧义？上下文中的真正含义是什么？

**操作方法**：
1. 找准关键词：在论题、结论和理由中标记核心术语
2. 检查有没有歧义：问"这个词是否可能有两种以上解读？"
3. 判定歧义：在上下文中，这个词的哪种含义最合理？
4. 字典定义≠情境含义：不要用字典定义替代上下文中的真正含义
5. 谁想说服你，谁就要负责解释清楚——如果作者未澄清歧义，这是论证的弱点

**适用材料类型**：全部

---

##### 步骤 5：假设

**核心问题**：作者未明说的价值观假设和描述性假设是什么？

**操作方法**：

寻找价值观假设：
1. 到哪儿找：在理由和结论之间的逻辑跳跃处
2. 价值观冲突和价值观假设：找出作者在两种冲突价值观中的选择
3. 把立论者的背景作为线索：作者所属组织、专业领域暗示什么价值倾向
4. 把可能发生的后果作为线索：如果采纳作者的结论，谁受益？谁受损？这暗示什么价值观？
5. 典型的价值观冲突：个人责任 vs 社会支持、自由 vs 安全、效率 vs 公平、传统 vs 进步

寻找描述性假设：
1. 描述性假设是关于"世界是什么样的"未明说信念
2. 常见的描述性假设：过去如此将来也如此、大多数人的选择是对的、权威来源是可靠的
3. 找到假设的线索：在理由和结论之间追问"还需要什么前提才能让理由支持结论？"

**适用材料类型**：论说类、社科类、科普类

---

##### 步骤 6：谬误

**核心问题**：论证是否依赖错误假设或逻辑谬误？

**操作方法**：
1. 从假设中寻找谬误：如果步骤5发现的假设不合理，论证可能包含谬误
2. 检查常见谬误清单：
   - 人身攻击（攻击人而非论证）
   - 滑坡谬误（假设某事会引发连锁反应却无证据）
   - 追求完美方案（因方案不完美而拒绝）
   - 偷换概念（悄悄改变关键术语含义）
   - 诉诸公众/权威/情感（用非理性方式说服）
   - 虚假两难（只给两个选择，实际有更多）
   - 乱扣帽子（用标签代替解释）
   - 循环论证（结论出现在理由中）
3. 警惕分散注意力的干扰：攻击稻草人、转移话题
4. 标记每个谬误的具体位置和类型

**适用材料类型**：论说类、学术类、商业类

---

##### 步骤 7：证据效力（一）——个人经历、典型案例、当事人证言、专家陈述

**核心问题**：这些证据类型可靠吗？有何局限？

**操作方法**：
1. 区分事实和见解：作者陈述的是可验证的事实，还是主观见解？
2. 逐类评估：
   - 个人经历：生动但代表性差，容易以偏概全
   - 典型案例：有说服力但可能经过挑选，检查是否为孤例
   - 当事人证言：可能受利益、记忆偏差、筛选影响
   - 专家意见：检查专家是否在自身领域内、是否有利益冲突、是否代表学界共识
3. 问：这个证据还可能被怎样解读？有没有反面证据被忽略？

**适用材料类型**：论说类、商业类、科普类

---

##### 步骤 8：证据效力（二）——个人观察、调查研究

**核心问题**：调查样本有代表性吗？测量方法可靠吗？

**操作方法**：
1. 评估个人观察：观察者是否有偏见？观察条件是否可控？
2. 评估调查/问卷：
    - 样本能否代表总体？（样本量、抽样方式、覆盖面）
    - **必须具体列出：样本包含了哪些人群/国家，遗漏了哪些。不能只写"覆盖面有限"，必须写出"覆盖了X国（均为中高收入），未覆盖Y国（低收入）"**
    - **如果书中有人群细分数据（不同群体的正确率/错误率差异），必须呈现**
    - 问卷措辞是否引导性？
   - 回答是否真实？（社会期望偏差）
3. 评估研究报告：
   - 是否经过同行评审？
   - 是否可重复验证？
   - 研究条件与实际场景的差异有多大？
4. 测量方法可靠吗？同一个现象用不同方法测量，结果是否一致？
5. 专家观点什么时候最可信：经过同行评审、领域内共识、无利益冲突、承认不确定性

**适用材料类型**：学术类、科普类

---

##### 步骤 9：替代原因

**核心问题**：是否存在其他可能的原因解释同一结果？

**操作方法**：
1. 寻找替代原因：问"还能怎么解释这个结果？"
2. 评估替代原因的说服力：哪个原因最简洁、最全面、最可验证？
3. 区分"唯一的原因"和"原因之一"：多数结果有多种共同起作用的原因
4. 警惕"确认偏误"：不要只寻找支持已有观点的原因
5. 混淆相关性和因果关系：A和B相关不代表A导致B
6. 混淆"在此之后"与"因此"：时间先后≠因果
7. 解释单个事件时，检查是否过度简化了原因

**适用材料类型**：论说类、社科类、科普类

---

##### 步骤 10：数据欺骗性

**核心问题**：数据来源、平均值、省略信息是否有误导？

**操作方法**：
1. 识别不知来历和有偏差的数据：数据从哪来？谁资助的研究？
2. 令人困惑的平均值：是均值还是中位数？极端值是否扭曲了平均？
3. 测量误差：数据的误差范围是多少？差异是否在误差范围内？
4. 错把一件事的结论用来证明另一件事：数据证明的是A，作者却用来证明B
5. 通过省略信息进行欺骗：对比的基线是什么？绝对数还是比例？缺少了什么参照？

**适用材料类型**：论说类、商业类、科普类

---

##### 步骤 11：省略信息

**核心问题**：哪些关键信息被遗漏？遗漏对结论有何影响？

**操作方法**：
1. 找到省略信息的益处：更完整的画面才能做出更可靠的判断
2. 不完整的论证在所难免：所有论证都省略了信息，关键是省略的是否重要
3. 帮你识别省略信息的问题：
   - 反面视角：反对者会说什么？
   - 缺失的数字：需要什么具体数据才能评估这个论证？
   - 长期影响：这个行动的潜在负面后果是什么？
   - 语境信息：这个数据在什么条件下成立？
4. 面对信息缺失的现实：有时必须在不完整信息下做判断，但要承认不确定性

**适用材料类型**：全部

---

##### 步骤 12：合理结论

**核心问题**：除了作者的结论，还有哪些合理的替代结论？

**操作方法**：
1. 警惕二分式思维：不要只看到"是/否"两个选项
2. 灰度思维：在两个极端之间寻找多种可能性
3. 条件的重要性：结论可能在不同条件下成立——"如果X，那么结论A；如果Y，那么结论B"
4. 解放思维：问"理由还能支持什么其他结论？"
5. 更多可能的结论，更多可能的自由选择：替代结论越多，决策空间越大

**适用材料类型**：论说类、学术类、科普类

---

##### 步骤 13：认知障碍

**核心问题**：作者或读者可能受到哪些认知偏见影响？

**操作方法**：逐项对照以下偏见清单：

**作者的偏见**：
- [ ] 晕轮效应：因作者某个特征（声望、外貌）而过度接受/拒绝其所有观点
- [ ] 信念固着：作者是否拒绝修正已有信念，只寻找支持性证据？
- [ ] 确认偏误：作者是否只呈现支持结论的证据，忽略反面证据？
- [ ] 可得性启发法：作者是否过度依赖最容易想到的信息，而非最相关的信息？
- [ ] 一厢情愿：作者是否因为希望某事为真就断言其为真？

**读者的偏见**：
- [ ] 正确提问带来的不快：读者是否因被质疑而不适，拒绝深入思考？
- [ ] 思考过快：读者是否依赖"系统1思维"（直觉）而非"系统2思维"（慢思考）？
- [ ] 刻板印象：读者是否因对作者/群体的刻板印象而预判结论？
- [ ] 信念固着：读者是否因已有信念而拒绝接受新证据？
- [ ] 确认偏误：读者是否只注意支持自己观点的分析，忽略反面分析？
- [ ] 可得性启发法：读者是否因最近看到的新闻/信息而过度评估某事的重要性？
- [ ] 近因效应：读者是否因最近读到的内容而过度影响判断？
- [ ] 答非所问：读者是否在回应时偷换了问题？
- [ ] 自我中心：读者是否以自己的经历为中心，无法理解他人的立场？
- [ ] 一厢情愿：读者是否因为希望某事为真而相信其为真？
- [ ] 奇迹式思维：读者是否在缺乏证据时依赖"奇迹"式解释？

**适用材料类型**：全部

### 阶段 4 — 精华提炼

从深度分析中提炼两部分内容，帮助读者内化：

1. **毋庸置疑的事实**：从分析中提取那些不依赖于理论框架、可独立验证的核心事实。标准：
    - 有权威数据支撑（联合国、世界银行等同行评审来源）
    - 实验可重复（如大猩猩测试）
    - 不依赖于作者的理论框架也成立
    - **归因范围与证据范围匹配：如果证据只覆盖了部分人群（如中高收入国家），事实描述必须限定范围，不能用"人类""所有人"等全称归因**

2. **可内化的方法论**：提取作者提出的、经过批判性思维检验后仍然成立的思维工具/方法。每个方法包含：
   - 方法名称
   - 核心操作（一句话说明怎么用）
   - 适用场景（什么时候该用这个方法）

### 阶段 5 — 练习场

基于分析结果生成5-10道实操练习题，帮助读者练习批判性思维技能。

格式：从原文摘取一段，要求读者用某步骤的操作方法进行分析。

示例："用步骤4的操作方法，找出以下段落中的歧义词：[摘录原文段落]"

### 阶段 6 — 纠错与验证

合并"反驳/支持"和"自我反思"为一个板块，形成完整的纠错与验证闭环：

1. **反驳路径**：基于分析，最有力的反驳切入点是什么？
2. **支持路径**：基于分析，最坚实的支撑点是什么？
3. **书评建议**：如果写书评，建议从哪个步骤切入？
4. **自我反思**：
   - 这本书挑战了我哪些原有信念？
   - 我是否也在犯作者指出的谬误？
   - 我的分析是否受到了确认偏误的影响？
   - 如果我站在作者的立场，最合理的辩护是什么？
5. **修正记录**：分析过程中的回溯修正

### 阶段 7 — 输出报告

将分析结果写入 `.reading/` 目录下，生成交互式 HTML 报告。

## 注意事项

- 分析应引用原文关键段落，追溯每个判断到具体出处
- 练习题应融入实操元素，从原文摘取段落让读者用具体操作方法分析
- 如果材料过长，优先分析用户指定的章节，其余简要概括
- 使用项目领域词汇（参考 `CONTEXT.md` 如果存在）
- 所有判断都是暂时的——后续步骤的发现可能要求修正前步判断，必须记录修正
- 作者论证链阶段严格按作者逻辑组织，不引入批判性思维框架视角
